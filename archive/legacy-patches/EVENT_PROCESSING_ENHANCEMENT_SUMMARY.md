# Enhanced API Server - Event Processing Capabilities

## 🎯 **Overview**
The API server has been significantly enhanced to handle comprehensive event-specific txt files generated by your n8n workflow. All event fields are now properly parsed, structured, and saved to Firebase.

---

## ✅ **Enhanced Features**

### **1. Comprehensive Event Field Support**
The API server now processes all event-specific fields:

#### **Core Event Fields:**
- `Event Title` → Main title and eventDetails.eventTitle
- `Event Description` → Main description and eventDetails.eventDescription  
- `Event Date` → eventDate field
- `Event Start Time` → eventTime and eventDetails.eventStartTime
- `Event End Time` → eventTimeEnd and eventDetails.eventEndTime
- `Event Duration` → eventDetails.eventDuration + anythingElseImportant

#### **Event Organization:**
- `Organizer` → eventDetails.organizer + anythingElseImportant
- `Event Format` → eventDetails.eventFormat + auto-sets remote flag
- `Event Type` → eventDetails.eventType + anythingElseImportant
- `Target Audience` → eventDetails.targetAudience + anythingElseImportant

#### **Logistics & Booking:**
- `Ticket Price` → eventDetails.ticketPrice + anythingElseImportant
- `Booking/Registration` → eventDetails.bookingRegistration + anythingElseImportant
- `Capacity` → eventDetails.capacity + anythingElseImportant
- `Venue Details` → eventDetails.venueDetails + anythingElseImportant

#### **Policies & Requirements:**
- `Refund Policy` → eventDetails.refundPolicy + anythingElseImportant
- `Contact Information` → eventDetails.contactInformation + anythingElseImportant
- `Special Requirements` → eventDetails.specialRequirements + anythingElseImportant

### **2. Intelligent Data Management**

#### **Smart Title Generation:**
- Uses Event Title if available
- Falls back to "{Industry} {Event Type}" format
- Defaults to "Creative Event" for events without specific titles

#### **Deadline Handling:**
- Uses explicit "Application deadline" if provided
- Falls back to Event Date if no specific deadline
- Supports both "Application Deadline" and "Application deadline" formats

#### **Remote Detection:**
- Automatically sets `remote: true` if Event Format contains "online"
- Maintains manual Remote field for override capability

## 🔧 **Technical Implementation**

### **Enhanced parseTextFile() Function:**
- **41 event-specific field handlers** added
- **Case-insensitive deadline parsing** (supports both formats)
- **Multi-line value support** for long descriptions
- **Intelligent field consolidation** in anythingElseImportant
- **Automatic remote flag detection** based on event format

### **Enhanced transformData() Function:**
- **Preserves all event details** in Firebase document
- **Conditional eventDetails object** (only added if event fields present)
- **Full backward compatibility** with existing opportunity format
- **Proper tag mapping** for all demographic fields

## 🎯 **Workflow Integration**

### **Complete Pipeline:**
1. **n8n Workflow** → Extracts comprehensive event data from web pages
2. **Saves txt file** → With all event fields in key: value format
3. **Google Script** → Detects file, reads content, sends to API Bridge
4. **Enhanced API Server** → Parses all event fields, maps demographics, saves to Firebase
5. **Master Portal** → Displays rich event information with all details
